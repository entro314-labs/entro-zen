title: Breakpoints
---

Breakpoints allow you to build responsive layouts by reacting to screen size changes.
There are 6 predefined breakpoint values, `xs`, `sm`, `md`, `lg`, `xl` and `default`.
Breakpoints are triggered once the screen width is greater than the specified width value.

<Example>
  <DataTable
    data={[
      { name: 'default', device: 'Phones (portrait)', width: '0px' },
      { name: 'xs', device: 'Phones (landscape)', width: '520px' },
      { name: 'sm', device: 'Tablets (portrait)', width: '728px' },
      { name: 'md', device: 'Tablets (landscape)', width: '1024px' },
      { name: 'lg', device: 'Laptops', width: '1280px' },
      { name: 'xl', device: 'Desktops', width: '1640px' },
    ]}
  >
    <DataColumn id="name" label="Size" />
    <DataColumn id="device" label="Device" />
    <DataColumn id="width" label="Width" />
  </DataTable>
</Example>

## Usage

Certain properties of layout components accept a `Responsive` object, which specifies what value
to apply at a given breakpoint.

For example, the [Heading](/docs/typeography/heading) component takes a `size` prop. If we pass in a
`Responsive` object, we can specify which sizes to display at different breakpoints.

<Example>
  <Heading
    size={{
      xs: '2',
      sm: '3',
      md: '4',
      lg: '5',
      xl: '6',
    }}
  >
    {`This text will shrink/grow as the screen resizes.`}
  </Heading>
</Example>

```jsx
<Heading
  size={{
    xs: '2',
    sm: '3',
    md: '4',
    lg: '5',
    xl: '6',
  }}
>
  {`This text will shrink/grow as the screen resizes.`}
</Heading>
```
